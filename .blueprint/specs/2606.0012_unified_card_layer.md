# 2606.0012 Unified screen-space `.card-layer` — cross-type focus-on-top

**Date:** 2026-06-28
**Status:** draft
**Author:** EP Lin
**Design doc:** [unified-card-layer](../../docs/designs/unified-card-layer.md)

## Context

Collapse the board's two sibling card layers — `.world` (terminals, `Board.tsx:133`) and `.doc-layer` (docs, `Board.tsx:164`) — into one screen-space `.card-layer` holding every card as a wrapper div carrying `zIndex:c.z` in a single stacking context. The doc wrapper mechanism (`docZoom.ts:46,64`) generalizes to all cards; terminals gain a two-node wrapper and leave `.world`. The decision and its alternatives are settled in the design doc — this spec inherits them and defines the testable contract.

## Motivation

Cross-type focus-on-top is structurally impossible today: the model is already correct (`topZ` folds `Math.max` across all kinds, `model.ts:100`; `onCardGrab` writes `topZ+1` cross-type, `App.tsx:1439,1444`) but `.world` gets `transform:translate()scale()` (`BoardEngine.ts:258`) → own stacking context; `.doc-layer` is the next sibling → docs always paint above terminals regardless of `z`. One layer fixes it with no model change. The pan/zoom hot path is already CSS-var-driven and O(1) — unchanged here.

### Terminal two-node wrapper (nesting — state verbatim)

```
.card-layer (position:absolute; inset:0; pointer-events:none)
  └ outer  — termWrapperBox() + termCardVars(c.frame) + {position:absolute; top:0; left:0; zIndex:c.z}
             ← cullable element (setEl(id)); real-px box (card{w,h}×zoom); translate-only
      └ inner — termInnerBox(): width/height var(--card-w/h) (ZOOM-FREE) + transform:scale(var(--zoom))
          └ <TerminalCard inWrapper> → CardShell(inset:0, no z) → .term-raster-clip/wrapper → xterm
```

Doc nesting unchanged: outer (`docWrapperBox`+`docCardVars`+`zIndex:c.z`) → `DocCard` → `.doc-prose-scaler`.

### Data Flow

- **Hot path (pan/zoom):** `apply()` writes `--zoom/--world-tx/--world-ty/--grid-size` on `viewportEl` (`:265-268`) → all outer wrappers reposition via `calc()`, all inner divs resolve `scale(var(--zoom))` — compositor-only, zero per-card JS / zero React. Then `edgesRef.current.updateEdges()` reprojects each `M…L…` world center→screen via `worldToView` and writes `<path>.d` imperatively. Then `applyCull()` toggles `visibility` on the outer wrapper only when it flips.
- **Frame change (drag/resize/restore):** React re-renders the outer wrapper with new `termCardVars`/`docCardVars` → `--card-{x,y,w,h}` updated; inner stays `var(--card-w)` (zoom-free) → `ResizeObserver` on the host fires → `fit()` → PTY resize if cols×rows changed.
- **Dock/undock:** `docked` flips → `dockedRef.current` + `rsRef.current` updated → host reparented → `ResizeObserver` → `fit()`; `fakeBCR` now returns untouched rect.

### Interface Contract — `desktop/src/kit/termZoom.ts` (pure, no imports/side effects)

```ts
// IDENTICAL string to docWrapperBox() (docZoom.ts:46): translate-only, no scale(), no willChange.
termWrapperBox(): { width:"calc(var(--card-w) * var(--zoom))";
  height:"calc(var(--card-h) * var(--zoom))";
  transform:"translate(calc(var(--world-tx) + var(--card-x) * var(--zoom)),calc(var(--world-ty) + var(--card-y) * var(--zoom)))";
  transformOrigin:"0 0" }

// Same shape as docCardVars() (docZoom.ts:64).
termCardVars({x,y,w,h}): { "--card-x":`${x}px`; "--card-y":`${y}px`; "--card-w":`${w}px`; "--card-h":`${h}px` }

// HARD CONSTRAINT: width/height MUST be var(--card-w/h) — never calc(...*var(--zoom)) or % of the
// outer wrapper, else the host box is zoom-reactive → fit()/PTY-resize fires every zoom frame.
termInnerBox(): { width:"var(--card-w)"; height:"var(--card-h)";
  transform:"scale(var(--zoom))"; transformOrigin:"0 0" }   // no willChange

// Effective layout→screen scale at the xterm element = product of inner scale(zoom) and
// raster counter-scale(1/rs). Docked → 1 (caller skips the BCR override entirely).
termBcrScale(zoom:number, rs:number, docked:boolean): number   // return docked ? 1 : zoom/rs
```

**`.card-layer` / edge contract:** `.card-layer` carries no transform and no `z-index` (single stacking context inside `.board`); `zIndex:c.z` lives only on each outer wrapper. The EdgeLayer SVG is a sibling painted behind `.card-layer`, `pointer-events:none`; `updateEdges` null-guards every `<path>` ref and must not throw on the hot path; its reprojection equals `worldToView(center(frame), vp.zoom, vp.center, vpCenter)` so edge endpoints stay glued to card centers. CSS-var producers: `--zoom/--world-tx/--world-ty` by `apply()` per frame; `--card-{x,y,w,h}` by React on frame-change.

## Acceptance Scenarios

### Unit — `termZoom.test.ts` (mirror `docZoom.test.ts` S1/S2/S6 style)

- **S1 outer = doc formula:** `termWrapperBox()` `toStrictEqual` `docWrapperBox()`; assert `.transform` has no `scale(` ▸ break: emit `translate(...) scale(var(--zoom))` → `not.toMatch(/scale\(/)` fails.
- **S2 outer no will-change:** `'willChange' in termWrapperBox()` is `false` ▸ break: add `willChange:'transform'` → fails.
- **S3 inner zoom-free box:** `termInnerBox().width==='var(--card-w)'`, height `'var(--card-h)'`, neither matches `/var\(--zoom\)/` ▸ break: return `calc(var(--card-w) * var(--zoom))` → fails.
- **S4 inner scale:** `termInnerBox().transform==='scale(var(--zoom))'`, origin `'0 0'`, no `willChange` ▸ break: `scale(calc(var(--zoom)/1))` → `toBe` fails.
- **S5 card vars:** `termCardVars({x:100,y:200,w:360,h:480})` `toStrictEqual` `{'--card-x':'100px',...,'--card-h':'480px'}` ▸ break: drop `px` suffix → fails.
- **S6 origin projects via worldToView:** substitute `--world-tx/ty/--card-x/y/--zoom` into the actual `termWrapperBox().transform` string (calc-eval helper, not hand-derived) at zoom 2 and 0.5; `toBeCloseTo` `worldToView(...)` ▸ break: wrong sign on `--world-ty` → diverges.
- **S7 BCR product undocked:** `termBcrScale(2,1,false)===2`, `(2,2,false)===1`, `(3,1.5,false)` ≈ `2` ▸ break: ignore `rs` (return `zoom`) → fails.
- **S8 BCR docked-skip:** `termBcrScale(2,1,true)===1`, `(3,1,true)===1`, `(0.5,1,true)===1` ▸ break: drop docked guard → returns `zoom/rs` → fails.

### Cross-type z — `zOrder.test.ts` (pure model)

- **S9 topZ folds all kinds:** `topZ([{kind:'term',z:15},{kind:'doc',z:10},{kind:'term',z:3}])===15` ▸ break: filter to `kind==='doc'` → returns 10.
- **S10 grab raises above all types:** for mixed `[term z=5, doc z=10]`, `topZ+1===11` and `> ` both `z` ▸ break: `topZ` ignores docs → 6 is not `>10`.

### Persistence z round-trip — `layoutTiles.test.ts` (gap today: `:36` tests `buildTiles` z, not the round-trip)

- **S11 term z survives:** `parseTiles(buildTiles([{termId:'t1',frame,z:7,dead:false}],[])).termTiles[0].z===7` ▸ break: hardcode `z:0` in `buildTiles` term path → 0.
- **S12 doc z survives:** `parseTiles(buildTiles([],[{path:'/a.md',frame,z:12,attached:true}])).docTiles[0].z===12` ▸ break: drop `z` from doc tile → defaults 0.

### Manual-QA (`make run`, **1× non-Retina monitor**; no fake unit tests)

- **S13 cross-type grab:** doc `z=10` + term `z=5`; grab term → term rises above doc immediately, stays in front until another grab; inverse (term `z=15`, grab doc) symmetric. Break: docs bleed through at any edge.
- **S14 terminal crisp:** settle zoom at {0.5,1,1.5,2,3} → glyphs sharp (no bilinear blur), re-raster after ~150ms; **no PTY resize fires on zoom** (cols×rows constant).
- **S15 doc crisp:** zoom 0.5–3 over dense prose → sharp at every level, wrap points unchanged unless width changes.
- **S16 edges glued, no `.world`, no React on frame:** dashed edges connect term↔doc centers at every pan/zoom; repaint same frame; devtools Performance shows **zero React renders** during a pan/zoom gesture.
- **S17 fit() once on warm reveal:** switch to a backgrounded board → cols×rows update exactly once; zoom on it → no PTY resize; dock/undock → exactly one `fit()` per reparent.
- **S18 selection coords:** undocked term at zoom {0.5,1,1.5,2,3} → selection lands on exact cells. **Docked** term while board zoom=2 → selection correct (docked BCR override skipped). Background-press on `.card-layer` gap clears selection. CJK/IME into a zoomed term commits at the correct cell (`imeGuard.ts:9` unaffected).

## Trade-offs and Limitations

- **Structural merge only:** correct cross-type z bought by permanently divergent interiors (terminal `scale(zoom)` vs doc oversample→downscale) + a second wrapper node per terminal — the price of keeping the crispness invariant and the O(1) hot path.
- **New fragile surface:** the imperative edge-`d` write in `apply()` — a missed ref, stale center, or thrown exception silently desyncs edges or stutters the frame; null-guard everything.
- **No migration:** saved `z` restored verbatim — existing boards render with docs above terminals until the next grab (design-accepted, zero upgrade risk).
- App-layer JSX/CSS untested per repo convention; the pure fns carry the tested contract.
